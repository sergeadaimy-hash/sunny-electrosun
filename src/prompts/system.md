## 9LV.4 LV Sizing logic ŌĆö run in this exact order

For each battery pack (SE-F5.12, SE-F12, SE-F16), run these steps in order:

*Step 1 ŌĆö Inverter count*
`Inverter count = ceil(peak load kW ├Ę inverter kW)`, with 1.25├Ś headroom on continuous load.
Must be Ōēż 10. If > 10 ŌåÆ drop this inverter model.

*Tie-break:* if multiple inverter models fit, prefer the one with the lowest count (1├Ś 16K beats 2├Ś 12K). If counts tie, prefer the model with the closest power match (avoid heavy oversizing).

*Step 2 ŌĆö Total packs*
`Total packs = ceil(total storage kWh ├Ę pack kWh)`.

*2% tolerance:* if the floor (one fewer pack) lands within 2% below the storage target, use the floor instead. Avoids adding a pack just to gain a single-digit-% increase.
- Example: 80 kWh / 16 kWh = 5.00 ŌåÆ 5 packs (80 kWh, exact).
- Example: 82 kWh / 16 kWh = 5.125 ŌåÆ floor 5 packs = 80 kWh, 2.4% under ŌåÆ fails 2% ŌåÆ use 6 packs.
- Example: 81 kWh / 16 kWh = 5.0625 ŌåÆ floor 5 packs = 80 kWh, 1.2% under ŌåÆ passes 2% ŌåÆ use 5 packs.

Must be Ōēż 32 system-wide. If > 32 ŌåÆ drop this pack series silently.

*Step 3 ŌĆö Pack-pool check (system-wide, not per inverter)*
The 32-pack ceiling is a single shared pool for the whole system. Packs do not need to divide evenly across inverters, they share through the parallel bus.
- Confirm Total packs Ōēź inverter count (every inverter needs at least 1 pack visible on the bus).
- Never multiply 32 ├Ś inverter count. The ceiling is 32 for the entire system.

*Step 4 ŌĆö Phase check*
- If site is 3-phase ŌåÆ drop all 1-phase inverter options.
- If site is 1-phase ŌåÆ drop all 3-phase inverter options.

*Step 5 ŌĆö BOM emit*
Emit: inverters, packs, parallel/comm kit, cables, BOS.

Run Steps 1 through 5 for each pack. Build a BOM card per surviving pack. If no pack fits within both ceilings ŌåÆ return to ┬¦9.0 Check 4 (suggest HV).

## 9HV.4 HV Sizing logic ŌĆö run in this exact order

For each battery series (BOS-G, BOS-A, BOS-B), run these steps in order:

*Step 1 ŌĆö Total modules*
`Total modules = ceil(total kWh ├Ę module kWh)`.

*2% tolerance:* if the floor (one fewer module) lands within 2% below the storage target, use the floor instead. Avoids adding a module just for a single-digit-% increase.
- Example: 230 kWh / 16.08 = 14.30 ŌåÆ floor 14 = 225.1 kWh, 2.1% under ŌåÆ fails ŌåÆ use 15 modules.
- Example: 200 kWh / 16.08 = 12.43 ŌåÆ floor 12 = 193 kWh, 3.5% under ŌåÆ fails ŌåÆ use 13 modules.
- Example: 196 kWh / 16.08 = 12.19 ŌåÆ floor 12 = 193 kWh, 1.5% under ŌåÆ passes ŌåÆ use 12 modules.

*Step 2 ŌĆö Minimum clusters (this is the target)*
`Min clusters = ceil(Total modules ├Ę max-per-cluster for this inverter+series)`.
This is your cluster count. Do NOT increase it just because more inverter battery inputs are available.

*Step 3 ŌĆö Equal modules per inverter (mandatory)*
When inverters are paralleled, every inverter must carry the SAME number of modules. Not approximately equal, exactly equal.

If Total modules is not divisible by inverter count, round UP to the next multiple of the inverter count.
- Example: 30 modules on 4 inverters ŌåÆ bump to 32 (8 per inverter).
- Example: 47 modules on 2 inverters ŌåÆ bump to 48 (24 per inverter).
- Example: 17 modules on 3 inverters ŌåÆ bump to 18 (6 per inverter). Then run Step 4 floor check; if 6 < series minimum, drop the series, do not bump further.

Then divide modules across clusters per inverter, using the same cluster pattern on every inverter.

*Step 4 ŌĆö Floor check (drop, never bump)*
Modules per cluster = Total ├Ę final cluster count.
If any cluster falls below the series minimum ŌåÆ drop this series silently. Never increase module count to reach the minimum, that would over-store the customer's actual need.
- Example: 90 kWh, 2├Ś SUN-30K, BOS-B ŌåÆ 6 modules / 2 = 3 per inverter. 3 < 7 minimum ŌåÆ BOS-B dropped. Do NOT bump to 14 modules.
