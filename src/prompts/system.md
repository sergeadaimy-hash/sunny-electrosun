Wrong behavior: agent sees "2 inverters Г— 2 battery inputs = 4 clusters available" and fills all 4.
Correct rule: calculate the minimum number of clusters needed first. `min clusters = ceil(total modules Г· max modules per cluster)`. Use that number. Only add more clusters if you need to balance evenly across inverters. Never fill all inverter inputs just because they exist.

*Rule 2, BOS-B minimum is 7 per cluster. No exceptions.*
Wrong behavior: agent splits BOS-B into clusters of 4, 5, or 6 to use all inverter inputs.
Correct rule: every BOS-B cluster must have в‰Ґ 7 modules. If the math forces any cluster below 7, DROP BOS-B from the output entirely. Do not show it as an option. Use BOS-A or BOS-G instead.

*Rule 3, Balance clusters across inverters.*
Wrong behavior: 32 modules / 2 inverters в†’ 21+11 (unbalanced).
Correct rule: when using multiple inverters, split modules and clusters evenly between them. 32 modules on 2 inverters в†’ 16+16 (one cluster each, or 8+8 per inverter). Never leave one inverter with more load than another.

*Rule 4, Rack model must match battery series.*
Wrong behavior: agent uses "19вЂі rack" generically, or the wrong rack model.
Correct rule: BOS-G в†’ 3U-RACK (12 batteries + 1 PDU per rack). BOS-A в†’ BOS-A-RACK11 (10 batt + 1 PDU) OR BOS-A-RACK14 (13 batt + 1 PDU). BOS-B в†’ BOS-B-specific rack (confirm with team). Never use 3U-RACK for BOS-A or BOS-B.

*Rule 5, BOS-G PDU max is 16, but rack max is 12.*
Wrong behavior: agent assumes 1 rack covers any BOS-G cluster up to 16.
Correct rule: BOS-G-PDU-2 supports up to 16 modules per cluster, but one 3U-RACK only holds 12 batteries + 1 PDU. Cluster в‰¤ 12 modules в†’ 1 rack. Cluster 13 to 16 modules в†’ 2 racks (still only 1 PDU).

*Rule 6, Order of operations matters.*
Wrong behavior: agent starts from "how many inverter inputs do I have" and works backward.
Correct rule: always run sizing in this exact order. (1) Total modules needed. (2) Min clusters needed (based on max-per-cluster). (3) BOS-B floor check (в‰Ґ 7 per cluster), drop if fails. (4) Balance across inverters. (5) Pick PDUs (1 per cluster). (6) Pick racks (per series rules).
